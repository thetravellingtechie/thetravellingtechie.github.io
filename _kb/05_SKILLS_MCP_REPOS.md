# Skills, Protocoles MCP & Architectures Requises

# 🛠️ Les Compétences Clefs (The Traveling Techie)

Pour implémenter ces offres, Salim (et ses agents IA) auront besoin des *skills* combinés suivants :
1.  **Architecture Cloud & IaC** : Expertise certifiée Terraform, Kubernetes (AKS, EKS), Réseau hybride (VPN/Direct Connect).
2.  **Cybersécurité ("Zero Trust")** : Bonnes pratiques RBAC (Role-Based Access Control), Okta/EntraID, politiques de moindre privilège AWS IAM. Audit de conformité ISO27001/SOC2.
3.  **Vibe Coding & Ingénierie de l'IA** : Maîtrise des LLMs (Claude 3.5 Sonnet, GPT-4, Llama 3), création de "System Prompts", intégration API (OpenAI, Anthropic).
4.  **Développement & Scripting Automatisation** : PowerShell, Python, Bash, Node.js. (Crucial pour créer les workflows vendus aux entreprises).
5.  **Produit et Go-To-Market** : Capacité à vulgariser la stratégie tech, à packager une offre, et à écrire des "landing pages" convertissantes (Copywriting Tech).

# 🔌 Les Serveurs MCP (Model Context Protocol)

Pour que nous (l'IA assistante) puissions interagir puissamment avec les environnements de vos futurs clients (ou vos propres projets de "Vibe Coding"), les MCP suivants seront essentiels :

### 1. **Serveur MCP - GitHub / GitLab**
- **Utilité :** Permet à l'agent de chercher dans le code de l'entreprise, de lire les PRs, d'analyser l'infrastructure (ex: lire les fichiers Terraform et trouver des vulnérabilités de sécurité avant le déploiement).
- **Rôle :** Indispensable pour l'audit "Cloud Transformation Masterplan".

### 2. **Serveur MCP - Filesystem (Local)**
- **Utilité :** Pour exécuter le "Vibe Coding" au quotidien. L'IA peut générer le code et le modifier directement sur votre machine.

### 3. **Serveur MCP - Slack / Teams (Communication)**
- **Utilité :** L'agent peut lire les discussions techniques pour générer de la documentation automatique ("Knowledge Graph RAG") ou pour avertir en cas de détection d'une faille de sécurité (FinOps Alerting).

### 4. **Serveur MCP - Notion / Jira**
- **Utilité :** Pour mapper la gestion de projet et l'aligner avec la feuille de route technologique. Extraction des tickets bloqués, génération de résumés de réunion (C-Level Tech Strategy).

### 5. **Serveur MCP - AWS / Azure / GCP CLI (Outil de Commandes Sécurisées)**
- **Utilité :** Permet à l'agent d'envoyer des commandes de consultation sur l'infrastructure de production via les APIs Cloud (ex: *`aws ec2 describe-instances`*) afin de réaliser un audit Zero Trust en temps réel et de détecter les anomalies de configuration ou les surcoûts FinOps non documentés.

# 📦 Les Outils et Repositories "Indispensables"

Si nous construisons ces 5 "Produits sur étagère" (ex: l'Agent Compliance, ou le FinOps Automator), voici où nous forkerons nos bases pour gagner du temps :

### 1. n8n (o Make.com)
- **Repository :** `n8n-io/n8n`
- **Pour quoi ?** L'Auto-Publishing Techie Pipeline. Workflow visuel, auto-hébergeable.

### 2. Dify ou Langflow
- **Repository :** `langgenius/dify` ou `logspace-ai/langflow`
- **Pour quoi ?** Créer des Agents IA (RAG de connaissance interne) que vous pourrez déployer (ex: le "CTO Knowledge Graph") ou packager en SaaS B2B.

### 3. Cursor ou VS Code
- **Utilité :** Environnement du "Vibe Coding" quotidien intégrant les bases de Claude/OpenAI avec le code.

### 4. Terraform AWS/Azure/Google "Best Practices" Modules
- **Repository :** `terraform-aws-modules` et equivalents.
- **Utilité :** Le socle pour vendre les "Architectures de référence Cloud Blueprint". Vous n'avez pas à réécrire complétement le code, l'agent IA peut assembler et adapter ces modules éprouvés aux besoins du client.

### 5. Open WebUI (ou Ollama)
- **Repository :** `open-webui/open-webui`
- **Utilité :** Interface très solide si l'État ou l'Enterprise veut faire tourner une IA locale souveraine (ex: Llama-3 en on-premise) en raison de contraintes réglementaires / RGPD strictes. Excellent pour vos audits gouvernementaux et vos projets de confidentialité (B2G).
